# 工作流 BPM/BPMN 符合性说明

先统一术语，再说明符合性。

---

## 术语约定

| 术语 | 含义 | 格式 |
|------|------|------|
| **流程方案** | 一条流程模板记录（TaktFlowScheme / takt_workflow_scheme） | 数据库记录 |
| **流程定义** | 描述流程结构的“图纸”（节点与连线） | **BPMN 2.0 XML**，存于方案实体的 **BpmnXml** 字段 |
| **流程实例** | 一次流程运行（TaktFlowInstance） | 数据库记录 |
| **运行期数据** | 实例 ID、当前节点、变量、表单、API 入参/出参 | **JSON** |

- **流程定义** 仅指 BPMN 2.0 XML（设计期）；**不用 JSON 表示流程定义**。
- **运行期数据** 用 JSON（实例、变量、API）；与“流程定义”区分开。

---

## 格式与职责（一句话）

| 阶段 | 内容 | 格式 | 谁用 |
|------|------|------|------|
| **设计期** | 流程定义（流程图结构） | **BPMN 2.0 XML** | 设计器产出、引擎解析（BpmnXml） |
| **运行期** | 实例、变量、任务、表单、API | **JSON** | 前端、API、业务逻辑 |

流程解析 = 只解析 **XML**（BpmnXml）。流程数据（实例/变量/接口）= **JSON**。

---

## 一、流程定义（设计期）

### 1.1 后端

- **实体**：TaktFlowScheme 存 BpmnXml（流程定义）、ProcessJson（遗留，引擎不读）。
- **执行**：BuildRuntime 仅根据 BpmnXml 调用 LoadFromBpmnXml；成功则用解析结果，否则 BuildDefaultGraph。不根据 ProcessJson 构建流程。
- **结论**：流程定义以 BPMN 2.0 XML 为准，符合 BPMN；ProcessJson 不参与。

### 1.2 前端

- 方案 CRUD、BPMN 设计器（bpmn-js）保存 BPMN 2.0 XML 到 BpmnXml。
- **结论**：流程定义由设计器产出 XML，符合 BPMN。

---

## 二、流程执行（运行期）

### 2.1 后端

- **依据**：仅 BpmnXml 解析结果（或默认图）；实例、变量、API 为 JSON。
- **结论**：定义=XML，数据=JSON，与上表一致。

### 2.2 前端

- 发起、审批、撤回、历史等通过 API 与后端交互；请求/响应为 JSON。
- **结论**：运行期数据为 JSON，无概念混淆。

---

## 三、符合性小结

| 维度 | 结论 |
|------|------|
| 流程定义格式 | BPMN 2.0 XML（BpmnXml）；不以 JSON 为流程定义。 |
| 运行期数据格式 | JSON（实例、变量、API）。 |
| 执行引擎 | 只解析 BpmnXml 构建图；ProcessJson 不参与。 |
| 平台完整度 | 定义存储 + 实例生命周期 + 按图流转已实现；挂起/恢复、超时、通知、待办认领、加签/减签、退回未实现。 |

详见 **工作流实现总览.md**。

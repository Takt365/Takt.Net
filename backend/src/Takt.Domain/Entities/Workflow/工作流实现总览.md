# 工作流实现总览

本文档描述 Takt 工作流在代码中的实际实现。**先明确术语，避免混淆。**

---

## 术语（必读）

| 术语 | 含义 | 格式/载体 |
|------|------|-----------|
| **流程方案** | 一条流程模板记录（表 takt_workflow_scheme，实体 TaktFlowScheme） | 数据库记录，含 ProcessKey、BpmnXml、ProcessJson 等字段 |
| **流程定义** | 描述“流程长什么样”的图纸，即节点与连线的结构 | **BPMN 2.0 XML**，在本项目中存于方案实体的 **BpmnXml** 字段 |
| **流程实例** | 一次流程运行（表 takt_workflow_instance） | 数据库记录，含 CurrentNodeId、InstanceStatus 等 |
| **运行期数据** | 实例 ID、当前节点、变量、表单数据、API 请求/响应 | **JSON**（与“流程定义”无关，仅指业务数据与接口数据） |

约定：

- **流程定义** 只指 BPMN 2.0 XML，**仅用于设计期**；引擎**只**解析 BpmnXml 构建可执行图，**不以 JSON 作为流程定义**。
- **运行期** 的实例、变量、API 等用 **JSON**；不说“流程定义用 JSON”。

---

## 一、数据层（实体与表）

| 实体 | 表名 | 说明 |
|------|------|------|
| **TaktFlowScheme** | takt_workflow_scheme | 流程方案：ProcessKey、ProcessName、**BpmnXml**（流程定义）、ProcessJson（遗留）、FormId/FormCode、ProcessStatus 等 |
| **TaktFlowInstance** | takt_workflow_instance | 流程实例：InstanceCode、SchemeId、BusinessKey、CurrentNodeId/CurrentNodeName、InstanceStatus 等 |
| **TaktFlowExecution** | takt_workflow_execution | 工作流流转历史：实例从开始到结束经历的所有节点、步骤和状态变化的完整记录 |
| **TaktFlowOperation** | takt_workflow_operation | 操作记录：每次发起/审批/撤回等一条，OperationType、NodeId/NodeName、Content/Comment |
| **TaktFlowForm** | takt_workflow_form | 流程表单元数据：FormCode、FormName、FormTemplate 等 |

业务表（如 TaktAnnouncement、TaktDocument）通过 **InstanceId** 关联流程实例；发起时回写 InstanceId，结束时按 InstanceId 更新业务状态。

---

## 二、从方案到可执行图（仅认 XML）

- **入口**：`TaktFlowInstanceService.BuildRuntime(TaktFlowScheme scheme)`（私有）。
- **逻辑**：
  1. 若 `scheme.BpmnXml` 非空：`runtime.LoadFromBpmnXml(scheme.BpmnXml, ...)` → `TaktBpmnParser.Parse(bpmnXml)` 解析 BPMN 2.0 XML，得到内存中的 Nodes + Lines，返回 runtime。
  2. 否则：`runtime.BuildDefaultGraph()`，生成默认图（Start→Approval→End/Rejected），并设置 ProcessKey/ProcessName。
- **不参与**：不读取 ProcessJson 构建流程；ProcessJson 仅为遗留字段。

可执行图结构：`TaktFlowRuntime` 内存中持有一份 **Nodes**（TaktFlowNode）和 **Lines**（TaktFlowLine），以及按 FromNodeId 索引的 **FromNodeLines**，用于根据当前节点与通过/驳回条件计算下一节点。

## 三、流程执行调用链（后端）

| 操作 | 入口 | 关键步骤 |
|------|------|----------|
| 创建并启动 | `CreateAsync(dto)` | 按 ProcessKey 取方案 → BuildRuntime(scheme) → 取首节点 → 写 TaktFlowInstance → 回写业务表 InstanceId → 写操作记录 |
| 审批 | `ApproveAsync(dto)` | 取实例与方案 → BuildRuntime(scheme) → GetNextNode(当前节点, Pass/Reject) → 更新实例节点/状态 → 写历史 → 若终态则更新业务状态与操作记录 |
| 撤回 | `RecallAsync(dto)` | 校验发起人、运行中、可撤回 → BuildRuntime → 当前节点改回 Start，状态=已撤回 → 写操作记录 |
| 节点指定/转办 | `NodeDesignateAsync(dto)` | 运行中 → 当前节点改为 dto 指定 → 写历史 |
| 撤销审批 | `UndoVerificationAsync(dto)` | 已完成/已终止 → 按历史回退上一节点，恢复运行中 → 写操作记录 |

“下一节点”逻辑**仅**来自 TaktFlowRuntime（即 BpmnXml 解析结果或默认图）。

## 四、API 与前端

| 后端 | 前端（takt.antd） |
|------|-------------------|
| TaktFlowSchemesController | views/workflow/scheme：方案列表与编辑；BPMN 设计器（bpmn-js）写 BpmnXml |
| TaktFlowInstancesController | views/workflow/instance：发起、审批、撤回、流转历史等 |
| TaktFlowFormsController | views/workflow/form：表单元数据 CRUD（无表单设计器） |

## 五、已实现与未实现

- **已实现**：流程方案存储（BpmnXml 为流程定义）；BPMN 2.0 XML 解析（TaktBpmnParser）驱动可执行图，无 BpmnXml 时默认图；实例创建/审批/撤回/转办/撤销审批；流转历史与操作记录；业务表 InstanceId 回写与状态更新；前端 BPMN 设计器写 BpmnXml。**挂起/恢复**（UpdateStatusAsync，受方案 IsSuspendable 控制）；**退回**（ReturnAsync，退回到上一节点或指定节点，受 IsReturnable 控制）；**超时与自动推进**（TaktFlowTimeoutHostedService 每 5 分钟扫描，TimeoutConfig JSON：Hours/Minutes + Pass）；**通知扩展点**（ITaktFlowNotificationService：实例发起/节点到达/结束/超时，默认空实现，可对接站内信/SignalR/邮件）；**待办列表与认领**（GetTodoListAsync、ClaimAsync，实例 AssigneeId 字段）；**加签/减签**（接口与 API 占位，暂抛“需任务与候选人模型”）。
- **未实现**：加签/减签具体逻辑（需任务与候选人模型）；表单设计器。通知具体发送逻辑（由 ITaktFlowNotificationService 实现类解析 NotificationConfig 后实现）。

**数据库**：流程实例表需有 `assignee_id`（当前处理人ID，认领后写入）；若使用 CodeFirst 且实体已含该字段，建表/迁移时会自动添加。

更多符合性说明见同目录 **BPM与BPMN符合性说明.md**。

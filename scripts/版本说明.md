# 统一版本号说明

前后端共用同一套版本号，格式为 **0.1.年.月+序号**，年月自动取当前日期，序号按月自增。

## 版本格式

| 部分   | 说明           | 示例   |
|--------|----------------|--------|
| 0.1    | 固定前缀       | 0.1    |
| 年     | 当前年份       | 2026   |
| 月     | 01~12（两位）  | 01、02、12 |
| 序号   | 3 位，按月自增 | 001、002 |

示例：`0.1.2026.02001` → `0.1.2026.02002`（同月自增）；跨月后为 `0.1.2026.03001`。

## 涉及文件

| 文件 | 作用 |
|------|------|
| 仓库根目录 `version.json` | 唯一版本源，存 `year`、`month`、`sequence`，由脚本读写 |
| `scripts/bump-version.mjs` | 自增并同步到后端、前端的 Node 脚本 |
| `backend/src/Directory.Build.props` | 后端版本：`Version`、`AssemblyVersion`、`FileVersion` |
| `frontend/takt.antd/package.json` | 前端 `version` 字段 |

## 使用方式

在**仓库根目录**执行：

```bash
# 自增版本并同步到前端、后端（发版时执行）
node scripts/bump-version.mjs
```

在 `frontend/takt.antd` 目录下也可：

```bash
npm run version:bump    # 自增并同步
npm run version:sync    # 仅同步不自增（按当前 version.json 写回前后端）
```

- **自增**：同月则序号 +1，跨月则月份更新、序号从 001 开始。
- **同步**：把 `version.json` 中的版本写入 `Directory.Build.props` 和 `package.json`，不改变序号。

## 后端版本约定

- `Version`：显示用，如 `0.1.2026.02002`。
- `AssemblyVersion` / `FileVersion`：4 段整数（如 `0.1.2026.2002`），其中最后一段 = 月×1000 + 序号，满足 .NET 程序集版本要求。

## 执行时机

仅在**需要发布新版本时**执行一次自增脚本即可；日常开发、编译、启动无需执行。
